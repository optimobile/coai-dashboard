# Dual-Brand Architecture Strategy: CouncilOf.AI + SafetyOf.AI

## Executive Summary

**CouncilOf.AI** = B2B backend infrastructure (enterprise/government)  
**SafetyOf.AI** = B2C consumer app (public/analysts)

This dual-brand strategy maximizes market coverage, clarifies positioning, and enables 3 distinct revenue streams with 6 conversion funnels.

## How They Work Together

### User Flow Example

1. **SafetyOf.AI user** reports AI hallucination via mobile app
2. Report triggers **CouncilOf.AI** 33-Agent Council voting
3. If human review needed → **CouncilOf.AI analyst** (trained via SafetyOf.AI) reviews
4. Decision made and published on both platforms
5. Enterprise (CouncilOf.AI customer) receives webhook notification
6. Public (SafetyOf.AI users) see transparency feed update

### Data Flywheel

```
SafetyOf.AI users report issues
    ↓
CouncilOf.AI processes via 33-Agent Council
    ↓
Analysts (trained via SafetyOf.AI) review
    ↓
Data improves AI models
    ↓
Better recommendations for enterprises
    ↓
More enterprises adopt CouncilOf.AI
    ↓
More analyst jobs created
    ↓
More SafetyOf.AI users sign up
    ↓
Cycle repeats (network effects)
```

## SafetyOf.AI Mobile App

### Core Features

1. **Report AI Issue**
   - Quick report submission (text, screenshot, URL)
   - Category selection (hallucination, bias, privacy, safety)
   - Severity rating (low/medium/high/critical)
   - Anonymous or public reporting option

2. **33-Agent Council Voting**
   - Real-time voting visualization
   - See which AI agents voted approve/reject/escalate
   - View consensus status (approved/rejected/escalated)
   - Read agent reasoning for each vote

3. **Analyst Review**
   - If escalated, human analyst reviews
   - See analyst decision and reasoning
   - Track resolution status
   - Receive notifications on report updates

4. **Transparency Feed**
   - Public feed of all reports and decisions
   - Filter by category, severity, status
   - Search by keyword or company
   - Share reports on social media

5. **Training & Certification**
   - Access all SafetyOf.AI courses
   - Take certification exams
   - View certificates and badges
   - Track learning progress

6. **Job Marketplace**
   - Browse available analyst cases
   - Accept case assignments
   - Submit decisions and earn money
   - Track earnings and performance

### User Flow

```
1. User opens SafetyOf.AI app
2. Sees trending AI safety issues on feed
3. Clicks "Report Issue" button
4. Fills out quick form (2 minutes)
5. Submits report
6. 33 AI agents vote (30 seconds)
7. If escalated → Analyst reviews (24 hours)
8. User receives notification of decision
9. Report published on transparency feed
10. User can share on social media
```

## Database Schema Updates

### New Tables

**loi_signups**
- id (primary key)
- email (unique)
- ecosystem (enum: 'jobs_training', 'enterprise', 'early_adopters')
- source (string: landing page URL)
- created_at (timestamp)
- converted (boolean: false by default)
- converted_at (timestamp, nullable)
- conversion_value (decimal, nullable)

**founding_members**
- id (primary key)
- user_id (foreign key to users)
- membership_number (unique, e.g., "FM-0001")
- purchased_at (timestamp)
- lifetime_access (boolean: true)
- equity_allocation (decimal, nullable)
- referral_code (unique string)

**referral_program**
- id (primary key)
- referrer_user_id (foreign key to users)
- referred_user_id (foreign key to users)
- referral_code (string)
- commission_rate (decimal: 0.10 for 10%)
- commission_earned (decimal)
- status (enum: 'pending', 'paid', 'cancelled')
- created_at (timestamp)

**conversion_tracking**
- id (primary key)
- loi_signup_id (foreign key to loi_signups)
- user_id (foreign key to users, nullable)
- event_type (enum: 'page_view', 'signup', 'purchase', 'referral')
- event_data (JSON)
- created_at (timestamp)

### Updated Tables

**users**
- Add `brand` field (enum: 'councilof.ai', 'safetyof.ai', 'both')
- Add `founding_member` field (boolean: false by default)
- Add `referral_code` field (unique string)

**subscriptions**
- Add `ecosystem` field (enum: 'jobs_training', 'enterprise', 'early_adopters')
